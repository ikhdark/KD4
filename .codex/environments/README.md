# Codex Environments

`setup.py` prepares declared ignored files for Codex-created worktrees. It
currently handles only `user.bazelrc`, because the checked-in `.bazelrc` imports
that local file.

- Keep copied paths explicit in `setup.py`; do not add broad directory copies or
  glob synchronization.
- Preserve `--dry-run` as a non-mutating preview and `--force` as intentional
  overwrite behavior.
- Do not hand-edit `environment.toml`; it is autogenerated.
- Do not copy generated, vendored, lockfile, build-output, or cache paths unless
  they are explicitly required for environment setup.
