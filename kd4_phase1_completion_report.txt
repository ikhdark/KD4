6. Collision-safe tool-name helpers -- KD4_BETTER_SKIP

Decision:
  No KD2 code was ported. KD2's helper additions have no production callers and
  still flatten structurally different names into colliding strings. KD4's
  structural ToolName identity, explicit boundary flattening, and existing MCP
  callable-name normalization are safer. Retaining KD4 avoids restoring an
  upstream-removed helper surface without a consumer.

1. No publish or Desktop restart was performed.

   This was explicitly requested. Desktop visibility and installed-binary
   runtime proof remain deferred until a later explicit publish instruction.

2. App-server TypeScript schema fixture baseline is already dirty.

   The complete app-server-protocol test run passed 260/261 tests. The one
   failure is the pre-existing generated fixture-set mismatch for these six
   unrelated Server* files:

     ServerBuildInfo.ts
     ServerCapabilities.ts
     ServerLocalWatermark.ts
     ServerPackageLayout.ts
     ServerRuntimeInfo.ts
     ServerRuntimeWarning.ts

   This Phase 1 work does not change protocol type shapes, and no generated
   schema output remains modified by these changes.

3. The current Wiring Guard Rust comment stripper treats the existing documented
   route text `item/autoApprovalReview/*` in v2/item.rs as a block-comment start.

   Therefore the guard cannot statically see later code in that file. No source
   workaround was added. The canonical command conversion instead has direct
   unit coverage, successful crate compilation, and manual closure evidence.

4. app-server-test-client `serve` remains a Unix-style launcher.

   It still uses `nohup sh -c` and `tail -f /dev/null`. Phase 1 makes PATH,
   temp-directory selection, and listener/process helpers portable; it does not
   claim that the `serve` launcher itself is Windows-native.

5. Destructive live taskkill/kill testing was not performed.

   Parser and command-selection behavior compile and are unit-tested where safe.
   The actual listener termination path was not exercised against an arbitrary
   live process during this source port.

6. POSIX-only lsof/kill branches were not executed on this Windows host.

   Their implementation is a narrow extraction of the prior KD4 behavior plus
   deterministic PID deduplication.
